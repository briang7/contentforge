# MCP Servers

The Model Context Protocol (MCP) is an open standard that allows Claude Code to connect to external tools and data sources. MCP servers extend Claude's capabilities beyond built-in tools, enabling integration with databases, APIs, internal systems, and specialized development tools.

## What Is MCP?

MCP follows a client-server architecture:

- **MCP Client**: Claude Code acts as the client, sending requests to servers
- **MCP Server**: A process that exposes tools, resources, or prompts via the MCP protocol
- **Transport**: Communication happens over stdio (local processes) or HTTP/SSE (remote servers)

When you configure an MCP server, its tools become available to Claude alongside the built-in tools. Claude can discover what tools a server offers and invoke them as needed.

## Configuring MCP Servers

MCP servers are configured in your settings files:

### Project-Level Configuration

```json
// .claude/settings.json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "DATABASE_URL": "postgresql://user:pass@localhost:5432/mydb"
      }
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allowed/dir"]
    }
  }
}
```

### User-Level Configuration

```json
// ~/.claude/settings.json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    }
  }
}
```

## Server Configuration Fields

| Field | Required | Description |
|---|---|---|
| `command` | Yes | The command to start the server |
| `args` | No | Array of arguments for the command |
| `env` | No | Environment variables passed to the server process |
| `cwd` | No | Working directory for the server process |

## Popular MCP Servers

### Database Servers

```json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "DATABASE_URL": "postgresql://localhost:5432/mydb"
      }
    },
    "sqlite": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-sqlite", "./database.db"]
    }
  }
}
```

These let Claude query your database directly, inspect schemas, and help with SQL queries.

### File System Server

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/home/user/projects"]
    }
  }
}
```

Provides sandboxed file system access through MCP rather than direct tool access.

### GitHub Server

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    }
  }
}
```

Enables operations on GitHub repositories, issues, and pull requests through the GitHub API.

### Puppeteer / Browser Server

```json
{
  "mcpServers": {
    "puppeteer": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-puppeteer"]
    }
  }
}
```

Allows Claude to control a browser for testing, scraping, or taking screenshots.

## Using MCP Tools in Conversations

Once configured, MCP tools appear naturally in Claude's tool set. You can reference them directly:

```
> Query the database for all users created in the last 7 days

> Take a screenshot of localhost:3000 using the browser

> Search our GitHub issues for bugs labeled "high-priority"
```

Claude automatically selects the appropriate MCP tool based on your request.

## Building Custom MCP Servers

You can build your own MCP server using the official SDK:

```bash
npm install @modelcontextprotocol/sdk
```

A minimal TypeScript MCP server:

```typescript
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const server = new McpServer({
  name: "my-custom-server",
  version: "1.0.0",
});

server.tool(
  "get_weather",
  "Get current weather for a city",
  { city: z.string().describe("City name") },
  async ({ city }) => {
    const response = await fetch(
      `https://api.weather.example.com/${city}`
    );
    const data = await response.json();
    return {
      content: [
        { type: "text", text: JSON.stringify(data, null, 2) },
      ],
    };
  }
);

const transport = new StdioServerTransport();
await server.connect(transport);
```

Register it in your settings:

```json
{
  "mcpServers": {
    "weather": {
      "command": "node",
      "args": ["./tools/weather-server.js"]
    }
  }
}
```

## Troubleshooting

- **Server not starting**: Check that the command is in your PATH and dependencies are installed
- **Tools not appearing**: Restart Claude Code after changing MCP configuration
- **Environment variables**: Use `${VAR_NAME}` syntax to reference shell environment variables
- **Permissions**: MCP tool invocations follow the same permission system as built-in tools
